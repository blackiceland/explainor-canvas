## Визуальный манифест Explainor Studio

### 1. Базовая эстетика

- **Стиль**: минимализм, много воздуха, простые геометрические формы (круг, квадрат, решетка, линии).
- **Настроение**: спокойное, "slow thinking", без визуального шума.
- **Назначение**: технический контент, объяснение систем и архитектур, короткие сфокусированные сцены.

### 2. Цветовая палитра

- **Фон**
  - **background-warm**: мягкий теплый беж — основной фон светлых сцен.
  - **background-dark**: почти черный фон — для инвертированных сцен и акцентных кадров.

- **Базовые цвета фигур**
  - **ink**: чистый черный для основных форм и текста на светлом фоне.
  - **paper**: почти белый / светлый беж для фигур на темном фоне.

- **Текст**
  - **text.primary**: высокий контраст к фону (почти черный на светлом, почти белый на темном).
  - **text.muted**: второстепенные подписи (серый, но с достаточным контрастом).

- **Акценты (использовать редко и осмысленно)**
  - **accent.blue**: один холодный синий для подчёркивания ключевых элементов и путей.
  - **accent.red**: теплый красный для предупреждений, ошибок, "опасных" участков.
  - **Правило**: на одном кадре не более одного акцентного цвета; акцент всегда подчинен смыслу.

### 3. Формы и композиция

- **Примитивы**
  - Круги: сущности, точки внимания.
  - Квадраты и прямоугольники: блоки, сервисы, контейнеры.
  - Решетки и паттерны: повторение, множество, абстракция данных.
  - Полукруги, дуги, линии: движение, поток, разделение.

- **Композиция**
  - Центрированная или по сетке (2×2, 3×3, линейные ряды).
  - Много свободного пространства вокруг групп элементов.
  - Каждый кадр должен читаться как законченный плакат.

### 4. Типографика

- **Шрифт**: моноширинный или аккуратный гротеск (например, JetBrains Mono или аналог).
- **Регистр**: UPPERCASE для коротких тезисов, смешанный регистр для основного текста.
- **Отступы**: увеличенный интерлиньяж и поля для ощущения "воздуха".

### 5. Применение в коде

- Все значения палитры должны быть отражены в `StandardTheme` в `motion-canvas/src/core/theme.ts`.
- Новые компоненты должны брать цвета не напрямую ("#ff0000"), а из темы (`theme.colors.*`).
- При добавлении новых сцен:
  - избегать "радуги" — максимум один акцентный цвет на кадр;
  - сначала думать о композиции (грид, центр), затем о цвете.
